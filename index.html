Webcraft

06 mei 2014, Hermans Jana, Weet-me-Nietjes

Een typisch voorbeeld waarbij de architectuur van software/hardware reguleert wat je ermee kan doen is twitter. In een bericht van twitter kan je maar 140 tekens gebruiken. Hiermee kunnen gebruikers enkel korte, to the point berichten schrijven en posten. Persoonlijk is dit zeer handig om mensen creatief te laten zijn als ze hun beknopte boodschap duidelijk over willen brengen. Dit wilt natuurlijk ook zeggen dat deze site niet gemaakt is voor het posten van brieven, lange monologen, ed.

Bij de te volgen tutorials ivm (.....) stond oa Git. Omdat ik al met Github bezig ben sinds februari, ben ik over Git niet zo positief. Voor mij was het, zelfs met tutorials erbij, eerder een manier om tijd te verspelen dan een positieve bijdrager tot een verbreding van mijn kennis. Maar, omdat het een tutorial moet zijn die 'niets voor mij' is, heb ik die van Git maar gekozen. Om nu echt te zeggen dat ik hier ook daadwerkelijk iets heb uit geleerd ... Op zich kan ik deze informatie misschien later ooit wel toepassen, maar momenteel zie ik het niet echt als nuttig.

Ook met CSS ben ik, onbewust, al bezig geweest. Om bepaalde elementen in te voeren van een andere site heb ik (met de linkermuisknop) kunnen kiezen voor 'elementen inspecteren', en zo heb ik de gewenste elementen (en hun code) kunnen doorvoeren in gitHub. Om nu, na deze laatste paar weken, terug te kijken op de 'code' die gebruikt werd voor een bepaald gewenst effect, besef ik ook wel dat ik dit nooit zelf had kunnen bedenken. Langs de andere kant, nu ik wat meer weet over het web en over coderen en het gebruik van bepaalde tools, is het bekomen van een gewenst effect veel makkelijker te bereiken door het gebruik van MarkdownPad. Om een voorbeeld te geven: de voordien vrij ingewikkelde code om een 'archives' te maken zoals bovenaan deze site, kan veel simpler gemaakt worden met behulp van MarkdownPad. in plaats van een ellelange code kun je gewoon met hyperlinken werken met wat gewone tekst tussen. Simple, denk ik nu, maar niet altijd even gemakkelijk voor een beginner.
